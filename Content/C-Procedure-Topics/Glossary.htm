Types of content testing glossary

    A
  • A/B or multivariate test
    Simplified A/B testing is used to show preferences (this or that). This type of test shows the customers' preferences and answers the question about what they would like to use. A/B testing allows targeting target your resources for maximum output, making as few modifications as possible.
  • Acceptance testing
    Acceptance testing is performed after system testing. It is the way of checking the business requirements. We perform it to make sure that the content is ready for delivery.
  • Accessibility
    • C
  • Card sorting
    Card Sorting is a tool for exploring the “Mental Model” of participants. It means the way they search for information, products, or solutions.
  • Content testing
    • E
  • English readability testing tools
  • Examples of accesibility testing tools
    • F
  • First click testing
    First click testing is a method for measuring the usability of a website, app, or design by finding out how easy it is to complete a given task. First click tests can be run using screenshots, sketches, wireframes, prototypes, and mock-ups. Using it we can test all the design processes.
  • Flesch-Kincaid Grade Level
    The Flesch-Kincaid Grade Level is equivalent to the US grade level of education. It shows the required education to be able to understand a text.
  • Flesch-Kincaid Reading Ease
    The higher the reading score, the easier a piece of text is to read. A lower score is easier. For example, a reading score of 60 to 70 means a grade level of 8-9 (a text should be understood by 13 to 15-years old). Reading Ease score conversion table translates the score into a grade level.
  • Functionality
    • G
  • Google trends
    The tool you can use to follow what people search for. Google Trends gives access to the sample of actual search requests made to Google. It's anonymized (no one is personally identified), categorized (determining the topic for a search), and aggregated (grouped).
  • Grammarly
    Editing app using artificial intelligence (AI) for proofreading. All you need to do is copy the text’s passage and approve or dismiss the changes.
    • H
  • Hemingway
    The app highlights lengthy, complex sentences and common errors. It uses colors to show the readability of the sentences.
  • Highlighter test
    The technique to allow us to understand the impact of what we are writing. We're learning how to make our content clear and simple. How to do that? We ask participants to read a printout of content and to underline things that made them feel more confident about the service in green, and things that made them feel less confident in red.
  • Human editor
    Also called the second pair of eyes. A real person to review your content. A human editor can get a sense of your meaning more precisely than a computer. A person can detect the tone of the human writer throughout the content.
    • I
  • Integration testing
    The next step after unit testing. This is the way to check cooperation between the inter-connected modules.
    • J
  • Jasnopis
    This is a polish language-based tool that helps to check if the passage of the text is easy or difficult to read and understand. Jasnopis’s basic functionalities are free and require neither additional software nor advanced computer skills and linguistic knowledge.
    • L
  • Lighthouse
    An open-source automatic tool for improving web apps' performance, quality, and correctness. When auditing a page, Lighthouse runs a barrage of tests against the page and then generates a report on how well the page did.
  • Logios
    This tool is checking if the passage of the text is easy or difficult to read and understand. You copy the text you want to paste to the window to analyze it.
    • N
  • Navigability
    • P
  • Polish readability testing tools
  • Proofreading
  • ProWritingAid
    With this app, you can discover the secret to polished and professional writing. Nine out of ten writers make the same mistakes. ProWritingAid gives you clear, easy steps to improve your writing so you can share your ideas with confidence
    • R
  • Readability
  • Regression testing
    This test checks if a code or a new functionality implementation doesn't disturb previous functionalities.
    • S
  • Sanity testing
    Type of regression testing. It involves running a set of regression tests. We make sure that a new fix didn’t affect our working functionality. It takes less time than regression testing.
  • SlickWrite
    SlickWrite is an application that makes it easy to check your writing for grammar errors, potential stylistic mistakes, and other features of interest.
  • Smoke testing
    Smoke testing checks if a new software build and its functionality are stable. During this step, we check a limited number of test cases. We want to make sure that a new build can go for precise testing. If the smoke test cases go well, this version of the software is ready for more testing.
  • Steps of functional testing
  • System testing
    System testing is the third step of functional testing. It goes after integration testing. During this type of test, we check if the application or webpage matches the specifications.
    • T
  • The Flesch–Kincaid readability tests
    This is a mathematical formula that tells you how easy content is to read. Scores range from 0 to 100. The higher the score, the easier the text is to read. These tests are designed to show how difficult a passage in English is to understand. There are two tests: the Flesch Reading-Ease, and the Flesch–Kincaid Grade Level.
  • The Wave Extention from WebAIM
    The tool is used to identify areas in which a site can improve according to WCAG standards. Wave generates reports with a list of issues and advice for what to change. The Wave provides visual feedback about the accessibility of web content by injecting icons and indicators into your page.
    • U
  • Unit testing
    Its role is to test a module or piece of code. It shows if it works properly and can be integrated with other modules.
  • Usability
  • Usability testing methods
    • W
  • Web Accessibility Checker
    This tool gives a possibility to find out if your website is accessible and compliant. The user can scan the website, identify accessibility issues, and get instructions on how to fix them.
  • Web Content Accessibility Guidelines (WCAG)
    The Web Content Accessibility Guidelines are part of a series of web accessibility guidelines. WCAG was published by the Web Accessibility Initiative of the World Wide Web Consortium (the international standards organization for the Internet).
  • WhiteSmoke
    English writing tool that provides grammar, spelling, punctuation, and style checks.
